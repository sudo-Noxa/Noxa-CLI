## Install

```bash
git clone https://github.com/sudo-Noxa/Noxa-CLI
cd Noxa-CLI
pip install -r requirements.txt

# optional: set ngrok token for internet access
export NGROK_AUTHTOKEN=your_token_here
```

**requirements.txt**

```
websockets   >=12.0
cryptography >=42.0
colorama     >=0.4
pyngrok      >=7.0   # optional
```

## Usage

**Start a server (you become the host):**

```bash
python chat.py --host

# custom port
python chat.py --host --port 9000
```

**Connect as a client:**

```bash
# LAN
python chat.py --connect ws://192.168.1.5:8765

# internet via ngrok
python chat.py --connect wss://xxxx.ngrok.io

# with a custom nickname
python chat.py --nick ghost --connect wss://xxxx.ngrok.io
```

## Commands

| Command | Description |
|---|---|
| `/help` | Show command reference |
| `/users` | List users currently online |
| `/dm <nick> <text>` | Send an encrypted direct message |
| `/group new <name> <n1> <n2>` | Create an encrypted group chat |
| `/group add <name> <nick>` | Add a user to an existing group |
| `/group <name> <text>` | Send a message to a group |
| `/kick <nick>` | Remove a user *(host only)* |

## How it works

| Layer | Mechanism |
|---|---|
| **Key exchange** | Ephemeral X25519 keypair per session. Shared DM keys derived via HKDF-SHA256. Keys never touch disk. |
| **Encryption** | AES-256-GCM with a random 96-bit nonce per message. AEAD rejects any tampered ciphertexts. |
| **Padding** | All messages padded to multiples of 512 bytes before encryption. Short and long messages are indistinguishable by size on the wire. |
| **Cover traffic** | Random encrypted dummy packets sent every 5–15 s. Observers cannot tell when real messages are sent. |
| **Send jitter** | 50–400 ms random delay before every real send. Breaks timing-correlation between typing and wire events. |
| **Group keys** | Symmetric group key generated by the creator, wrapped individually for each member via ephemeral X25519. Server never sees the key. |

## Security model

| Layer | Mechanism | Status |
|---|---|---|
| Message contents | AES-256-GCM E2E — server sees only ciphertext | ✅ Protected |
| Message size | 512-byte block padding hides true length | ✅ Protected |
| Send timing | Jitter + cover traffic obscure real send events | ✅ Mitigated |
| Session keys | Ephemeral X25519 — rotate every session | ✅ Ephemeral |
| Message history | Not written to disk — memory only | ✅ No persistence |
| IP address | Visible to host — use Tor/VPN to hide | ⚠️ Exposed |
| Nickname | Sent plaintext on join, visible to all peers | ⚠️ Plaintext |
| Who talks to whom | Host sees connection metadata | ⚠️ Host-visible |
| Forward secrecy | No ratchet — single DM key per session | ❌ Not implemented |

> **Not anonymous by default.** Noxa protects message *contents*, not your network identity.  
> Your IP address is visible to the host. For real anonymity, run over **Tor** or a trusted VPN.
